今天在Martin Fowler的博客上看到一篇文章[Fragments: April 2](https://martinfowler.com/fragments/2026-04-02.html)，提到了一个新的概念：Intent Debt（意图债）。

这篇文章区分了技术债（Technical Debt）、认知债（Cognitive Debt）和意图债（Intent Debt）。广义上讲，其实可以统一归为技术债，但是在ai编成成为主流的时代，意图债这个概念的提出是非常有意义的。

这里有个简单的对比：
| Debt type | Lives in | Accumulates when | Limits |
|-----------|----------|------------------|--------|
| Technical debt | Code | Implementation shortcuts compromise future changeability | How the system can change |
| Cognitive debt | People | Shared understanding erodes faster than it's replenished | How teams can _reason_ about change |
| **Intent debt** | **Artifacts** | **Goals and constraints aren't captured or go stale** | **Whether the system still reflects what you meant to build** |
