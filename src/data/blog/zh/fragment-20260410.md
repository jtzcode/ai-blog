今天看到[这篇文章](https://arxiv.org/abs/2604.06268)，与 Agent 的强化学习训练相关，挺有意思。

论文研究的是 multi-turn LLM agents 的强化学习训练。在这类训练里，agent 要边想边做动作，还要和环境闭环交互。

作者指出，现有训练监控通常把 `entropy` 当成“reasoning 稳定性”的代理指标，但 `entropy` 只能告诉你：**同一个输入下，输出是否多样**；它不能告诉你：**不同输入之间，reasoning 是否真的被输入区分开了**。

于是就会出现一种很危险的假象：模型输出看起来仍然丰富、流畅、像在认真思考，但其实已经**越来越不看输入**了。

换句话说，如果模型只是学会了一套**通用模板**，比如“先分析、再计划、最后执行”这类表面上丰富的 boilerplate，它在 `entropy` 上未必难看，但在 agent 场景里会越来越不可靠。

当研究者看 entropy 时，他们常常在监控一种风险：模型是不是**越来越只会一个模板，越来越少探索，变得过度自信、过度集中**。

但是这一版是针对同一个输入而言的；对于不同类型的输入，模型理应展现出按照不同结构、模板的推理，而不仅仅是同一个模板的不同变体。

因此，作者把 reasoning 质量拆成两个维度：

- 条件熵 $H(Z \mid X)$：同一个输入下，reasoning 的多样性
- 互信息 $I(X; Z)$：reasoning 对输入的依赖程度，也就是"输入可区分性"

基于这两个轴，论文给了四种状态：

Diverse Reasoning：高条件熵、高互信息。既多样，又真的随输入变化。
**Template Collapse**：高条件熵、低互信息。看起来多样，但其实不看输入。
Compressed Reasoning：低条件熵、高互信息。很简短、很确定，但仍然对输入敏感。
Low-Entropy Collapse：低条件熵、低互信息。彻底塌掉了，既死板又不看输入。

这里最有价值的是第二种，即体现了**表面多样实际没有反映输入变化**的问题。

想法很好，但是想衡量这个「互信息」，不那么容易，因为 reasoning 是高维的 token 序列。

于是作者设计了一组 `MI proxies`。核心直觉是：如果一段 reasoning 真是由某个输入生成的，那么把这段 reasoning 拿去和一批候选输入做“匹配打分”时，应该更容易找回它真正对应的输入。

它本质上把“reasoning 是否依赖输入”转成了一个**检索**问题。

进一步，作者还分析了为什么会出现 template collapse 的现象，以及提出解决方案，并进行了实验证明。这部分理论性较强，感兴趣的同学可以研读原文。

通俗的理解问题原因就是：

训练时，不是每道题都能给模型提供同样清晰的学习信号。很多题不同尝试之间的奖励差别很小，模型就学不到“这题该怎么**因题而异**地思考”；

解决思路就是：

既然不是所有题都“值得学”，那每轮训练前就先挑那些**更能看出好坏差别的题来更新模型**。

论文用一个很简单的信号来做这件事：看同一道题不同 rollout 的奖励波动大不大；波动越大，说明这题更能提供有效学习信号。

总之，这还是一个训练数据的问题，强化学习需要的数据要求更高。大模型的训练过程，还是有很多未知的机制，需要通过不断的分析和实验来揭示。
